# Product Brief: Bmadcalculator

<!-- Content will be appended sequentially through collaborative workflow steps -->

## Executive Summary

Bmadcalculator is a dedicated calculator app built specifically for grade 3 children (ages 8-9). Standard calculator apps overwhelm young learners with buttons and functions they don't understand and don't need. Bmadcalculator solves this by stripping the calculator down to exactly what a grade 3 student requires — addition, subtraction, multiplication, and division — presented in a simple, toy-like interface designed for small hands and developing minds.

---

## Core Vision

### Problem Statement

Grade 3 children find standard calculator apps confusing and overwhelming. There are too many buttons — decimal points, memory functions, percentage keys, sign toggles — none of which are relevant to grade 3 mathematics. When a child sees an unfamiliar button and accidentally presses it, the result is confusion, frustration, and lost confidence.

### Problem Impact

A confused child stops using the tool and either gives up, guesses, or asks for constant adult help. The calculator — intended to assist learning — becomes a source of anxiety rather than empowerment. Teachers and parents are forced to supervise every interaction rather than letting children work independently.

### Why Existing Solutions Fall Short

Every mainstream calculator app (phone calculators, Google calculator, etc.) is designed for adults. They prioritise feature completeness over clarity. No major calculator app has been purpose-built for the grade 3 curriculum — they are all general-purpose tools handed to children who are not their intended audience.

### Proposed Solution

Bmadcalculator is a radically simplified calculator containing only what a grade 3 child needs: the numbers 0–9, four operation buttons (+, -, ×, ÷), an equals button, and a clear button. Nothing else. The interface is designed with toy-like clarity — large buttons, high contrast, distinct shapes for each operator, and a persistent equation display that shows the full calculation ("8 + 3 =") so children can self-check before pressing equals. Every interaction is immediate, satisfying, and impossible to get wrong.

### Key Differentiators

- **Radical simplicity by design** — fewer buttons is a feature, not a limitation
- **Full equation always visible** — children see what they've entered and can self-correct
- **Built for grade 3 only** — not a dumbed-down adult calculator, a purpose-built child's tool
- **Toy-like UI** — large, shaped, high-contrast buttons designed for small fingers and developing readers
- **Emotionally safe** — forgiving "oops" clear button, satisfying answer reveal, no scary functionality

## Target Users

### Primary Users

**Persona: Jamie, Age 8 — Grade 3 Student**

Jamie is 8 years old and in grade 3. After school, Jamie sits at the kitchen table or on the couch with a tablet, working through maths homework. A parent is nearby — cooking dinner, folding laundry, or sitting at the same table — available to help but not hovering over every calculation.

Jamie can read reasonably well but finds symbol-heavy interfaces confusing. When Jamie picks up a standard calculator app and sees rows of unfamiliar buttons, the instinct is to either hand the tablet back to the parent ("I don't know how to use it") or press buttons randomly hoping for the right answer. Neither outcome builds confidence or independence.

**What Jamie needs:**
- An interface so obvious it requires zero explanation
- Buttons that are impossible to accidentally misuse
- Immediate, clear feedback that confirms the right thing happened
- To feel capable and independent, not confused and reliant

**Jamie's "aha!" moment:** Completing a homework calculation correctly, on the first try, without asking for help.

## MVP Scope

### Core Features

The MVP delivers one thing exceptionally well: a calculator that a grade 3 child can use correctly and confidently, on a tablet, at home, without help.

**1. Number Input (0–9)**
Large, clearly labelled digit buttons sized for small fingers on a tablet touchscreen. Each press registers immediately with visible feedback.

**2. Four Operation Buttons (+, -, ×, ÷)**
Distinct from number buttons through shape and colour — not just labels. One operator selected at a time, clearly highlighted when active.

**3. Equals Button**
Prominent, larger than all other buttons. Pressing it triggers a satisfying animated answer reveal — the payoff moment of every calculation.

**4. Clear / "Oops" Button**
A forgiving, friendly clear button that removes the last entry rather than wiping everything. Styled to feel safe to press, not scary.

**5. Full Equation Display**
A persistent display showing the complete running equation ("8 + 3 =") as the child builds it, so they can self-check before pressing equals. Resets cleanly after each completed calculation.

**6. Web App (No Download Required)**
Accessible via any modern browser on a tablet. No installation, no account, no login. Alex bookmarks the URL — Jamie opens it and starts calculating.

**7. Tablet-Optimised Layout**
Designed specifically for tablet screen sizes in both portrait and landscape. Touch targets meet minimum accessibility sizing standards. High contrast, large text throughout.

---

### Out of Scope for MVP

Deliberately excluded to keep the MVP focused and shippable:

- Decimal point button
- Memory functions (M+, MR, MC)
- Percentage or sign-toggle buttons
- Sound effects or animations beyond the answer reveal
- Object-based / visual number entry
- Themes or customisation
- Classroom display mode
- Parent check-in / usage log
- User accounts or data storage
- Any form of scoring, rewards, or gamification

---

### MVP Success Criteria

The MVP is successful when:

- A grade 3 child opens it on a tablet and completes a calculation without adult guidance on the first attempt
- A parent bookmarks it and returns to the same URL in a subsequent homework session
- Zero bug reports related to incorrect calculation results
- The interface loads and functions reliably on a tablet browser
